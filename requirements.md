# 牌类游戏平台产品需求文档

## 一、项目概述

### 1.1 项目背景
本项目旨在打造一款多玩法牌类游戏平台，首期以「炸金花」作为切入点，通过最小可行产品（MVP）快速验证用户需求、留存能力和变现模型，后续逐步扩展至德州扑克、牛牛、百家乐等牌类游戏，形成稳定、可扩展的游戏平台。

### 1.2 产品定位
- **定位**：碎片化时间的线上娱乐牌类游戏平台
- **目标用户**：18-40岁，熟悉牌类玩法、追求刺激与博弈乐趣的用户
- **核心诉求**：
  - 上手快
  - 节奏快
  - 强情绪反馈
  - 可持续成长与竞争

### 1.3 产品目标
- 30天内跑通：注册 → 游戏 → 消耗 → 回流 → 复玩
- 验证核心指标：
  - 次日留存 ≥ 25%
  - 单局时长 30-90 秒
  - 金币回流率（抽水）稳定

## 二、整体产品结构

### 2.1 产品架构
```
牌类游戏平台
├── 统一大厅
│   ├── 用户信息 / 金币余额
│   ├── 游戏入口（炸金花 / 预告玩法）
│   └── 活动 / 公告
│
├── 游戏模块
│   ├── 炸金花（V1）
│   ├── 德州扑克（V2）
│   └── 牛牛 / 百家乐（V3）
│
├── 公共系统
│   ├── 账号系统
│   ├── 金币 & 经济系统
│   ├── 商城 / 充值
│   ├── 任务 / 活动
│   └── 风控 / 日志系统
```

### 2.2 设计原则
- 游戏模块化，可插拔
- 公共系统一次建设，多玩法复用
- 规则与 UI 分离，便于扩展

## 三、首发玩法：炸金花（完整方案）

### 3.1 玩法规则（简化版）
- 使用 52 张牌（无大小王）
- 每人 3 张牌
- 牌型大小：豹子 > 同花顺 > 同花 > 顺子 > 对子 > 单张
- 支持操作：
  - 看牌
  - 跟注
  - 加注（固定档位）
  - 弃牌
  - 比牌

### 3.2 桌型与场次
| 场次 | 入场金币 | 单注 | 抽水 |
|------|----------|------|------|
| 新手场 | 1,000 | 50 | 5% |
| 普通场 | 10,000 | 500 | 8% |
| 高级场 | 100,000 | 5,000 | 10% |

### 3.3 游戏流程
1. 匹配 / 入桌
2. 发牌（暗牌）
3. 回合轮流操作
4. 比牌 / 弃牌
5. 结算
6. 自动进入下一局

## 四、核心系统设计

### 4.1 账号系统
- 游客登录
- 手机号登录（可选）
- 同设备防多开

### 4.2 金币与经济系统
- 初始赠送金币
- 每局抽水（系统回收）
- 破产补助（每日限次）
- 金币用途：
  - 入场
  - 跟注 / 加注
  - 商城购买

### 4.3 商城系统
- 金币充值包
- 首充礼包
- VIP（月卡）

### 4.4 机器人系统（关键）
- 自动补桌
- 多级 AI（保守 / 激进 / 随机）
- 可控胜率区间
- 新手保护逻辑

### 4.5 风控与反作弊
- 行为异常检测
- 输赢异常统计
- 高频操作限制
- 日志全量记录

## 五、盈利模式设计

### 5.1 主要变现方式
- 虚拟金币充值
- VIP 特权（抽水减免 / 专属表情）
- 高级场门票

### 5.2 辅助变现
- 破产礼包
- 活动限时礼包
- 成就奖励反向刺激

## 六、技术实现方案（概要）

### 6.1 客户端
- Web / H5
- Phaser / Pixi
- WebSocket 通信

### 6.2 服务端
- Node.js
- WebSocket 房间模型
- 服务端发牌 / 比牌

### 6.3 核心模块
```
GameServer
├── RoomManager
├── GameEngine（炸金花规则）
├── PlayerState
├── RobotAI
├── EconomyService
└── AntiCheat
```

## 七、版本规划

### V1（验证期）
- 炸金花完整可玩
- 金币闭环
- 机器人系统

### V2（增长期）
- 排行榜
- 好友 / 私人房
- 活动系统

### V3（平台期）
- 新牌类玩法上线
- 跨玩法资产体系

## 八、核心指标（KPI）
- 注册 → 对局转化率
- 单局时长
- 次日 / 7 日留存
- ARPU
- 金币回收率

## 九、风险与应对

| 风险 | 应对方案 |
|------|----------|
| 留存低 | 调整节奏 / 新手保护 |
| 金币通胀 | 动态抽水 / 消耗点 |
| 冷启动 | 机器人填充 |
| 作弊 | 行为风控 |

## 十、总结

本项目以炸金花为切入点，采用平台化、模块化设计，优先验证玩法可行性与盈利模型，在风险可控的前提下逐步扩展牌类矩阵，最终形成稳定、可持续的牌类游戏平台。
